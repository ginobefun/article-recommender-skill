# Article Recommender - Claude AI 文章推荐技能

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Version](https://img.shields.io/badge/version-2.0.1-blue.svg)](https://github.com/yourusername/article-recommender-skill)

一个专业的 Claude AI 技能包，为 BestBlogs.dev 周刊生成三版本文章推荐语（标准版、精炼版、批判性评论版），支持中英文双语输出。

[English](README.md)

## 🌟 核心特性

通过 **智能多版本推荐** 提升内容策展体验：

### 三个策略版本
- 📋 **推荐版本** - 全面而简洁的概述（150-200 中文字符，80-120 英文单词）
- 🔍 **精炼简洁版本** - 直接到位的核心要点（80-120 中文字符，50-80 英文单词）
- 💬 **个人评论版本** - 客观的批判性分析与专业洞察（180-250 中文字符，100-150 英文单词）

### 多领域支持
- 🤖 **AI** - 关注技术深度、模型局限、实际应用
- 📱 **产品** - 强调用户价值、实践经验、资源要求
- 💼 **商业** - 处理幸存者偏差、情境因素、规模假设
- 💻 **编程** - 评估复杂度权衡、学习曲线、维护成本

### 多形式覆盖
- 📄 **文章** - 标准文本内容分析
- 🎥 **视频** - 包含时长和时间投入分析
- 🎙️ **播客** - 收听体验和信噪比评估
- 🐦 **推文/推文串** - 快速洞察，警惕过度简化
- 📑 **学术论文** - 研究质量和可重现性评估

## 🎯 技能功能

本技能提供系统化的工作流程来生成专业推荐语：

1. **分析输入** - 提取关键信息，识别内容领域和形式
2. **应用领域知识** - 使用领域特定和形式特定的策略
3. **生成三个版本** - 创建标准版、精炼版和批判性评论版
4. **双语输出** - 为每个版本提供自然的中文和英文
5. **质量检查** - 确保准确性、格式和语言自然度

## 📊 三版本框架

### 版本一：推荐版本
- **目的**: 全面概述，不冗长
- **风格**: 客观、专业、平衡
- **内容**: 核心价值 + 关键洞察 + 目标受众契合度
- **使用场景**: 大多数周刊读者的默认推荐

### 版本二：精炼简洁版本
- **目的**: 快速传递信息
- **风格**: 直接、高效、无废话
- **内容**: 只保留核心信息
- **使用场景**: 快速浏览或空间有限的场景

### 版本三：个人评论版本
- **目的**: 客观的批判性分析，有深度
- **风格**: 专业、直接、建设性
- **内容**: 批判性观察 + 值得思考的问题 + 实践背景
- **使用场景**: 展现专业判断，鼓励批判性思考

**版本三的关键特性**：
- 指出潜在的局限性和不足
- 提出值得讨论的问题
- 基于客观分析，而非个人偏好
- 保持建设性和专业的语气
- 识别可能不普遍成立的假设

## 🚀 安装方法

### 前置要求

- Claude 桌面应用或 Claude 网页版 (claude.ai)
- Claude Pro 订阅（推荐，以获得最佳体验）

### 安装步骤

1. **下载** 技能文件：[article-recommender.skill](article-recommender.skill)

2. **导入到 Claude：**

   **桌面应用：**
   - 打开 Claude 桌面应用
   - 点击右上角头像
   - 选择 Settings → Skills
   - 点击 "Import Skill"
   - 选择下载的 `.skill` 文件

   **网页版：**
   - 访问 [claude.ai](https://claude.ai)
   - 点击右下角设置图标
   - 进入 "Skills" 标签页
   - 点击 "Import" 按钮
   - 上传 `.skill` 文件

3. **开始使用！** 当你要求 Claude 编写推荐语时，技能会自动触发。

## 💡 使用指南

### 自动触发

当你使用以下任何表达时，技能会自动激活：

**中文触发词：**
- "帮我编写推荐理由"
- "生成推荐语"
- "写一个推荐"
- "为这篇文章写推荐"

**英文触发词：**
- "write a recommendation"
- "generate article recommendation"
- "help me recommend this article"
- "write recommendations for"

### 基础用法

直接提供文章信息并要求推荐：

```
请为这篇文章写推荐语：[粘贴内容或摘要]
```

### 完整用法（提供详细上下文）

如果你有文章的完整信息（比如使用 deep-reading-analyst 分析过的文章）：

```
帮我编写这篇文章的推荐理由，作为每周精选内容期刊给读者展示：

----------
标题: [文章标题]
来源: [文章来源]
一句话总结: [总结]

主要观点:
- [观点 1]
- [观点 2]
- [观点 3]

核心金句:
[关键引用]

全文内容:
[文章正文]
----------
```

### 指定内容类型

帮助技能更好地理解你的内容：

```
请为这个 40 分钟的 AI 技术演讲写推荐语
```

### 请求调整

自定义特定版本：

```
请写推荐语，但版本三的批判性可以更强一些
```

## 🌐 多形式处理

### 文章（标准文本）
- 强调核心论点和关键要点
- 突出实用价值和适用性

### 视频
- **标注时长**和所需时间投入
- 强调视觉/演示价值
- **批判性考虑**: 是否有配套资料？能否跳过部分内容？

### 播客
- **标注时长**和收听体验
- 突出讨论主题和嘉宾专业度
- **批判性考虑**: 信噪比、播放速度/口音

### 推文/推文串
- 快速捕捉核心洞察
- 承认格式固有的简洁性
- **批判性考虑**: 过度简化风险、缺失的细微差别

### 学术论文
- 解释研究问题和贡献
- 引用方法论和发现
- **批判性考虑**: 可重现性、评估局限性、理论与实践差距

## 💡 技巧和最佳实践

### ✅ 推荐做法

1. **提供上下文** - 分享文章摘要、关键观点、目标受众
2. **指定格式** - 说明是视频、播客、推文串等
3. **注明时长** - 对于视频/播客，包含时间长度
4. **指示领域** - 提及是否跨多个领域（如 AI + 产品）
5. **请求调整** - 要求特定的语气或深度变化

### ❌ 避免做法

1. 不要只提供链接而没有上下文
2. 不要期望技能能猜测目标受众
3. 不要忽视版本三中提出的批判性观点
4. 不要在未审核准确性的情况下使用推荐语

## 🔍 高级特性

### 智能术语处理

技能自动：
- **保留英文**用于通用技术术语（API、SDK、LLM、RAG、MVP、CI/CD）
- **使用中文**用于有自然中文对应的概念（机器学习、用户体验、产品迭代）
- **保持一致性**跨所有三个版本
- **正确间距**在中英文/数字之间

示例：
- ✓ LLM 应用开发
- ✓ API 接口设计
- ✓ 机器学习模型优化
- ✓ 持续集成 CI/CD 流程

### 质量保证

自动检查：
- ✅ 地道的中文表达（避免翻译腔）
- ✅ 自然的英文写作（避免中式英语）
- ✅ 每种语言的正确标点
- ✅ 基于原文的准确信息
- ✅ 平衡的语气，不浮夸
- ✅ 正确的 Markdown 格式

## ❓ 常见问题

**Q：技能能否直接分析网页链接？**
A：可以！当你提供 URL 时，Claude 可以获取网页内容。

**Q：能处理非英文内容吗？**
A：可以！无论源语言是什么，都会生成双语输出（中文 + 英文）。

**Q：我可以只要求一个或两个版本吗？**
A：当然可以！只需指定你需要哪个版本即可。

**Q：领域检测的准确性如何？**
A：技能使用内容分析来识别领域。如果需要，你可以明确指定。

**Q：我能自定义版本三的批判深度吗？**
A：可以！根据需要请求"更批判"或"更温和"的反馈。

**Q：版本 2.0 有什么新功能？**
A：新增多领域支持（4 个领域）、多形式处理（5 种形式）、客观批判性评论框架和全面的术语处理指南。
